COFROX THIRD-PARTY LICENSE INVENTORY
Updated: March 27, 2026

This file lists the third-party components that are actually referenced by the
current Cofrox source tree and, when applicable, the optional external tool
bundle that can be produced from a local Tools folder. It is not a full SBOM.
It is a practical redistribution and compliance inventory for release
engineering.

========================================================================
REDISTRIBUTED BY DEFAULT VS OPTIONAL TOOL BUNDLE
========================================================================

1. FFmpeg
   Version:
   - Optional local bundle reports: N-123619-g44ad73031d-20260325
   License:
   - Effective license for the bundled binary: GPL-3.0-or-later
   Short description:
   - Media transcoding, probing, filters, and audio/video conversion engine.
   Attribution:
   - Copyright (c) 2000-2026 the FFmpeg developers
   Why this matters:
   - The developer download script currently pulls a binary built with --enable-gpl, --enable-version3,
     --enable-libx264, --enable-libx265, --enable-libxvid, and other options
     that move the binary out of the default LGPL posture.
   Obligations:
   - Preserve FFmpeg notices and GPL license text.
   - Provide the exact corresponding source code for the exact binary shipped.
   - Preserve build instructions/configuration for the shipped binary.
   - Do not describe this bundled binary as "LGPL-only" in product materials.
   Risk:
   - Highest legal risk in the optional offline tool bundle. Replace with a self-built LGPL-only
     FFmpeg if the release goal is simple MIT distribution and Microsoft Store
     packaging without GPL redistribution overhead.

========================================================================
RECOMMENDED PROJECT LICENSE STRATEGY
========================================================================

- Use MIT for the Cofrox application code.
- Keep copyleft obligations scoped to separately redistributed third-party
  executables and their notices.
- Before Store publication, replace the current FFmpeg bundle with a verified
  LGPL-only build or be prepared to ship full GPLv3 corresponding source for
  the FFmpeg component.
- Strongly consider removing the bundled Pandoc binary from the default Store
  package unless you are comfortable with GPL redistribution obligations for
  that component.
